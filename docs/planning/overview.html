3 핵심 기능

1

식재료 관리

집에 있는 식재료를 종류명만 등록 (그람수 없음). AI가 재료별 적정 섭취량과 요리법을 알아서 추천. 추가·삭제·목록 조회 가능.

2

신체 지표 추적

핵심 수치 3가지만 기록: 몸무게 · 체지방률 · 골격근량. 주기적으로 새 측정값 추가, 변화 추이 그래프 시각화. AI가 변화 추이를 참고해 식단 방향 조정.

3

AI 식단 추천 채팅 (메인)

텍스트 채팅 인터페이스. AI가 질문하면서 A/B/C 선택지를 메시지로 제시, 사용자는 선택지 또는 자유 텍스트로 답변. 식재료·신체지표·목표·이전 이력 모두 context로 반영. 요리법·섭취량·영양 포인트까지 함께 제안.

4

목표(지향성) 설정 — 복수 선택

체중 감량
체중 증량
체중 유지
벌크업
린매스
건강 관리
5

이전 메뉴 중복 방지

추천된 식단 이력을 로컬에 저장, AI context에 최근 N일 식단 포함. 추후 강제 중복 방지 로직 고도화 예정.

4 사용자 흐름

첫 실행
  └─ 온보딩 (성별 선택 → 목표 설정 → 첫 신체 지표)
       └─ 식단 채팅 탭 (메인 화면)
            ├─ AI와 대화로 오늘 식단 결정
            ├─ 식재료 탭에서 재료 관리
            ├─ 신체지표 탭에서 측정값 입력
            └─ 설정 탭에서 목표 수정

5 AI 연동 방식 (구현 시 결정)

방식설명상태
A 시스템 프롬프트 주입 스킬·에이전트 MD 내용을 system prompt로 삽입. 심플한 구현. 후보
B Function Calling 스킬·에이전트를 OpenAI function으로 변환, AI가 직접 호출. 동적이고 확장성 높음. 후보

공통 원칙: AI provider 추상화 레이어 설계 → 추후 Claude 교체 가능

## 1. 앱 개요

| 항목 | 내용 |
|------|------|
| 앱 이름 | gugbab-health |
| 성격 | 1인용 개인화 식단 관리 PWA |
| 목적 | 집에 있는 식재료와 내 신체 상태를 기반으로 AI가 오늘 먹을 식단을 추천해준다 |

## 2. 핵심 문제 정의

- 매일 "오늘 뭐 먹지?"를 고민하는 시간 낭비
- 냉장고에 있는 재료를 모르고 새로 사는 비효율
- 다이어트·벌크업 등 목표는 있지만 어떻게 먹어야 할지 막막함
- 같은 메뉴를 반복해서 질리는 문제

---

## 3. 핵심 기능

### 3-1. 식재료 관리
- 현재 집에 있는 식재료를 **종류명만** 등록 (그람수 없음)
- AI가 재료별 적정 섭취량과 요리법을 알아서 추천
- 재료 추가 / 삭제 / 목록 조회

### 3-2. 신체 지표 추적
- 핵심 수치만 기록: **몸무게 · 체지방률 · 골격근량**
- 주기적으로 새 측정값 추가 입력 (시계열)
- 변화 추이 그래프로 시각화
- AI가 변화 추이를 참고해 식단 방향 조정

### 3-3. AI 식단 추천 채팅 (메인 기능)
- 텍스트 채팅 인터페이스 (지금 이 대화와 동일한 UX)
- AI가 질문하면서 A/B/C 선택지를 메시지로 제시
- 사용자는 "A" 또는 자유 텍스트로 답변 가능
- 등록된 식재료 + 신체 지표 + 목표를 context로 전달
- 이전 식단 이력 참고해 메뉴 중복 최소화
- AI가 요리 방법, 적정 섭취량, 영양 포인트까지 함께 제안

### 3-4. 목표(지향성) 설정
사용자가 복수 선택 가능:
- 체중 감량
- 체중 증량
- 체중 유지
- 벌크업 (근육 증가)
- 린매스 (체지방 감소 + 근육 유지)
- 건강 관리 (혈당·콜레스테롤 등)

### 3-5. 이전 메뉴 중복 방지
- 추천된 식단 이력을 로컬에 저장
- AI context에 최근 N일 식단 포함
- 추후: 특정 기간 내 중복 강제 방지 로직 추가 예정

## 4. 사용자 흐름

```
첫 실행
  └─ 온보딩 (성별·목표 설정)
       └─ 식단 채팅 탭 (메인)
            ├─ 오늘 식단 추천받기
            ├─ 식재료 탭에서 재료 관리
            ├─ 신체지표 탭에서 측정값 입력
            └─ 설정 탭에서 목표 수정
```

## 5. AI 연동 방식 (결정 보류)

실제 구현 시점에 A/B 중 선택:

| 방식 | 설명 |
|------|------|
| **A) 시스템 프롬프트 주입** | 스킬·에이전트 MD 내용을 system prompt로 삽입, 심플 |
| **B) Function Calling** | 스킬·에이전트를 OpenAI function으로 변환, AI가 직접 호출 |

**공통 원칙**: AI provider 추상화 레이어 설계 → 추후 Claude 교체 가능

Vercel 재배포 버튼 안내에서 숨은 메뉴(⋯ → Redeploy) 경로만 기억에 의존해 반복 안내 → 사용자가 화면의 "Create Deployment" 버튼을 스스로 찾음. "다큐먼트 보고 확인하고 말해라, 대충 말로 때우지 마라"는 강한 질책을 받음.

**Why:** 사용자는 답의 근거가 검증됐는지를 매우 중시함. 추측성 안내가 한 번 틀리면 신뢰가 크게 깎이고, 같은 안내의 반복은 더 크게 화나게 함.

**How to apply:**
- 외부 도구 UI 조작법·사실 주장은 공식 문서(WebFetch)나 실측(curl·코드·번들 확인)으로 **검증한 뒤** 답한다 — 프로젝트 info-verification 규칙과 동일 기준.
- 안내가 한 번이라도 안 통하면 같은 경로를 반복하지 말고, 화면에 뭐가 보이는지 먼저 묻고 가능한 대안을 전부 나열한다.
- "안 된다"는 보고를 받으면 반박 전에 재현·실측부터 (모델 시트 z-index 건: 사용자가 옳았음 — 스크린샷 요청이 가장 빨랐다. 스크린샷은 프로젝트 루트에 놓아주기도 함).
- 증상 설명이 모호하면 원인 유형을 표로 나눠 어떤 것을 확인했고 어떤 것은 못 했는지 구분해 보고한다.
